# Unit File

The Unit File section provides a comprehensive view of all units and their associated residents in the building.

## Page Header

- **Title**: "Units"
- **Building Selector**: Dropdown to select building (e.g., "Bond")
- **Manage Staff**: Button to manage staff members
- **Search All Buildings**: Checkbox to search across all buildings
- **Status Filter**: Dropdown filter (e.g., "Active") with clear button

## Data Table

### Display Controls
- **Show Entries**: Configurable number of entries per page (default: 50)
- **Search**: Text search field to filter the table

### Table Columns (sortable)
| Column | Description |
|--------|-------------|
| Unit# | Unit number (e.g., 701, 702, 703) |
| Visit Unit | Edit/visit icon link to the unit details |
| First Name | Resident's first name |
| Visit User | Edit/visit icon link to the user profile |
| Last Name | Resident's last name |
| Groups | Resident type: Owner, Tenant, Offsite Owner |
| Email | Resident email address |
| Phone | Resident phone number |
| Front Desk Instructions | Special instructions for front desk staff |
| Active | Active status (Yes/No) |
| Parking | Assigned parking spot(s) (e.g., B-10, B-11) |

### Resident Group Types
- **Owner**: Unit owner residing in the unit
- **Tenant**: Renting resident
- **Offsite Owner**: Owner who does not reside in the unit

## Features
- All columns are sortable (indicated by sort arrows)
- Each unit and user has a clickable edit/visit icon for detailed view
- Multiple residents can be associated with a single unit
- Supports filtering by building, status, and search text
- Units are numbered sequentially (701, 702, 703, 704, 801, 802, etc.)
